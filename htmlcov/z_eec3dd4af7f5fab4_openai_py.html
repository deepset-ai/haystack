53@component 

54class OpenAIChatGenerator: 

55 """ 

56 Completes chats using OpenAI's large language models (LLMs). 

57 

58 It works with the gpt-4 and gpt-5 series models and supports streaming responses 

59 from OpenAI API. It uses [ChatMessage](https://docs.haystack.deepset.ai/docs/chatmessage) 

60 format in input and output. 

61 

62 You can customize how the text is generated by passing parameters to the 

63 OpenAI API. Use the `**generation_kwargs` argument when you initialize 

64 the component or when you run it. Any parameter that works with 

65 `openai.ChatCompletion.create` will work here too. 

66 

67 For details on OpenAI API parameters, see 

68 [OpenAI documentation](https://platform.openai.com/docs/api-reference/chat). 

69 

70 ### Usage example 

71 ```python 

72 from haystack.components.generators.chat import OpenAIChatGenerator 

73 from haystack.dataclasses import ChatMessage 

74 

75 messages = [ChatMessage.from_user("What's Natural Language Processing?")] 

76 

77 client = OpenAIChatGenerator() 

78 response = client.run(messages) 

79 print(response) 

80 ``` 

81 Output: 

82 ``` 

83 {'replies': 

84 [ChatMessage(_role=<ChatRole.ASSISTANT: 'assistant'>, _content= 

85 [TextContent(text="Natural Language Processing (NLP) is a branch of artificial intelligence 

86 that focuses on enabling computers to understand, interpret, and generate human language in 

87 a way that is meaningful and useful.")], 

88 _name=None, 

89 _meta={'model': 'gpt-5-mini', 'index': 0, 'finish_reason': 'stop', 

90 'usage': {'prompt_tokens': 15, 'completion_tokens': 36, 'total_tokens': 51}}) 

91 ] 

92 } 

93 ``` 

94 """

132 Creates an instance of OpenAIChatGenerator. Unless specified otherwise in `model`, uses OpenAI's gpt-5-mini

134 Before initializing the component, you can set the 'OPENAI_TIMEOUT' and 'OPENAI_MAX_RETRIES' 

135 environment variables to override the `timeout` and `max_retries` parameters respectively 

136 in the OpenAI client. 

137 

138 :param api_key: The OpenAI API key. 

139 You can set it with an environment variable `OPENAI_API_KEY`, or pass with this parameter 

140 during initialization. 

141 :param model: The name of the model to use. 

142 :param streaming_callback: A callback function that is called when a new token is received from the stream. 

143 The callback function accepts [StreamingChunk](https://docs.haystack.deepset.ai/docs/data-classes#streamingchunk) 

144 as an argument. 

145 :param api_base_url: An optional base URL. 

146 :param organization: Your organization ID, defaults to `None`. See 

147 [production best practices](https://platform.openai.com/docs/guides/production-best-practices/setting-up-your-organization). 

148 :param generation_kwargs: Other parameters to use for the model. These parameters are sent directly to 

149 the OpenAI endpoint. See OpenAI [documentation](https://platform.openai.com/docs/api-reference/chat) for 

150 more details. 

151 Some of the supported parameters: 

152 - `max_completion_tokens`: An upper bound for the number of tokens that can be generated for a completion, 

153 including visible output tokens and reasoning tokens. 

154 - `temperature`: What sampling temperature to use. Higher values mean the model will take more risks. 

155 Try 0.9 for more creative applications and 0 (argmax sampling) for ones with a well-defined answer. 

156 - `top_p`: An alternative to sampling with temperature, called nucleus sampling, where the model 

157 considers the results of the tokens with top_p probability mass. For example, 0.1 means only the tokens 

158 comprising the top 10% probability mass are considered. 

159 - `n`: How many completions to generate for each prompt. For example, if the LLM gets 3 prompts and n is 2, 

160 it will generate two completions for each of the three prompts, ending up with 6 completions in total. 

161 - `stop`: One or more sequences after which the LLM should stop generating tokens. 

162 - `presence_penalty`: What penalty to apply if a token is already present at all. Bigger values mean 

163 the model will be less likely to repeat the same token in the text. 

164 - `frequency_penalty`: What penalty to apply if a token has already been generated in the text. 

165 Bigger values mean the model will be less likely to repeat the same token in the text. 

166 - `logit_bias`: Add a logit bias to specific tokens. The keys of the dictionary are tokens, and the 

167 values are the bias to add to that token. 

168 - `response_format`: A JSON schema or a Pydantic model that enforces the structure of the model's response. 

169 If provided, the output will always be validated against this 

170 format (unless the model returns a tool call). 

171 For details, see the [OpenAI Structured Outputs documentation](https://platform.openai.com/docs/guides/structured-outputs).

455 def _prepare_api_call( # noqa: PLR0913 

456 self, 

457 *, 

458 messages: list[ChatMessage], 

459 streaming_callback: StreamingCallbackT | None = None, 

460 generation_kwargs: dict[str, Any] | None = None, 

461 tools: ToolsType | None = None, 

462 tools_strict: bool | None = None, 

463 ) -> dict[str, Any]: 

464 # update generation kwargs by merging with the generation kwargs passed to the run method 

465 generation_kwargs = {**self.generation_kwargs, **(generation_kwargs or {})} 

466 

467 is_streaming = streaming_callback is not None 

468 num_responses = generation_kwargs.pop("n", 1) 

469 

470 if is_streaming and num_responses > 1: 

471 raise ValueError("Cannot stream multiple responses, please set n=1.") 

472 response_format = generation_kwargs.pop("response_format", None) 

473 

474 # adapt ChatMessage(s) to the format expected by the OpenAI API 

475 openai_formatted_messages = [message.to_openai_dict_format() for message in messages] 

476 

477 flattened_tools = flatten_tools_or_toolsets(tools or self.tools) 

478 tools_strict = tools_strict if tools_strict is not None else self.tools_strict 

479 _check_duplicate_tool_names(flattened_tools) 

480 

481 openai_tools = {} 

482 if flattened_tools: 

483 tool_definitions = [] 

484 for t in flattened_tools: 

485 function_spec = {**t.tool_spec} 

486 if tools_strict: 

487 function_spec["strict"] = True 

488 function_spec["parameters"]["additionalProperties"] = False 

489 tool_definitions.append({"type": "function", "function": function_spec}) 

490 openai_tools = {"tools": tool_definitions} 

491 

492 base_args = { 

493 "model": self.model, 

494 "messages": openai_formatted_messages, 

495 "n": num_responses, 

496 **openai_tools, 

497 **generation_kwargs, 

498 } 

499 

500 if response_format and not is_streaming: 

501 # for structured outputs without streaming, we use openai's parse endpoint 

502 # Note: `stream` cannot be passed to chat.completions.parse 

503 # we pass a key `openai_endpoint` as a hint to the run method to use the parse endpoint 

504 # this key will be removed before the API call is made 

505 return {**base_args, "response_format": response_format, "openai_endpoint": "parse"} 

506 

507 # for structured outputs with streaming, we use openai's create endpoint 

508 # we pass a key `openai_endpoint` as a hint to the run method to use the create endpoint 

509 # this key will be removed before the API call is made 

510 final_args = {**base_args, "stream": is_streaming, "openai_endpoint": "create"} 

511 

512 # We only set the response_format parameter if it's not None since None is not a valid value in the API. 

513 if response_format: 

514 final_args["response_format"] = response_format 

515 return final_args

517 def _handle_stream_response(self, chat_completion: Stream, callback: SyncStreamingCallbackT) -> list[ChatMessage]: 

518 component_info = ComponentInfo.from_component(self) 

519 chunks: list[StreamingChunk] = [] 

520 for chunk in chat_completion: 

521 assert len(chunk.choices) <= 1, "Streaming responses should have at most one choice." 

522 chunk_delta = _convert_chat_completion_chunk_to_streaming_chunk( 

523 chunk=chunk, previous_chunks=chunks, component_info=component_info 

524 ) 

525 chunks.append(chunk_delta) 

526 callback(chunk_delta) 

527 return [_convert_streaming_chunks_to_chat_message(chunks=chunks)]

529 async def _handle_async_stream_response( 

530 self, chat_completion: AsyncStream, callback: AsyncStreamingCallbackT 

531 ) -> list[ChatMessage]: 

532 component_info = ComponentInfo.from_component(self) 

533 chunks: list[StreamingChunk] = [] 

534 try: 

535 async for chunk in chat_completion: 

536 assert len(chunk.choices) <= 1, "Streaming responses should have at most one choice." 

537 chunk_delta = _convert_chat_completion_chunk_to_streaming_chunk( 

538 chunk=chunk, previous_chunks=chunks, component_info=component_info 

539 ) 

540 chunks.append(chunk_delta) 

541 await callback(chunk_delta) 

542 

543 except asyncio.CancelledError: 

544 await asyncio.shield(chat_completion.close()) 

545 # close the stream when task is cancelled 

546 # asyncio.shield ensures the close operation completes 

547 # https://docs.python.org/3/library/asyncio-task.html#shielding-from-cancellation 

548 raise # Re-raise to propagate cancellation 

549 

550 return [_convert_streaming_chunks_to_chat_message(chunks=chunks)]

572 """ 

573 Converts the non-streaming response from the OpenAI API to a ChatMessage. 

574 

575 :param completion: The completion returned by the OpenAI API. 

576 :param choice: The choice returned by the OpenAI API. 

577 :return: The ChatMessage. 

578 """ 

579 message: ChatCompletionMessage | ParsedChatCompletionMessage = choice.message 

580 text = message.content 

581 tool_calls = [] 

582 if message.tool_calls: 

583 # we currently only support function tools (not custom tools) 

584 # https://platform.openai.com/docs/guides/function-calling#custom-tools 

585 openai_tool_calls = [tc for tc in message.tool_calls if not isinstance(tc, ChatCompletionMessageCustomToolCall)] 

586 for openai_tc in openai_tool_calls: 

587 arguments_str = openai_tc.function.arguments 

588 try: 

589 arguments = json.loads(arguments_str) 

590 tool_calls.append(ToolCall(id=openai_tc.id, tool_name=openai_tc.function.name, arguments=arguments)) 

591 except json.JSONDecodeError: 

592 logger.warning( 

593 "OpenAI returned a malformed JSON string for tool call arguments. This tool call " 

594 "will be skipped. To always generate a valid JSON, set `tools_strict` to `True`. " 

595 "Tool call ID: {_id}, Tool name: {_name}, Arguments: {_arguments}", 

596 _id=openai_tc.id, 

597 _name=openai_tc.function.name, 

598 _arguments=arguments_str, 

599 ) 

600 

601 logprobs = _serialize_object(choice.logprobs) if choice.logprobs else None 

602 meta = { 

603 "model": completion.model, 

604 "index": choice.index, 

605 "finish_reason": choice.finish_reason, 

606 "usage": _serialize_object(completion.usage), 

607 } 

608 if logprobs: 

609 meta["logprobs"] = logprobs 

610 

611 return ChatMessage.from_assistant(text=text, tool_calls=tool_calls, meta=meta)

614def _convert_chat_completion_chunk_to_streaming_chunk( 

615 chunk: ChatCompletionChunk, previous_chunks: list[StreamingChunk], component_info: ComponentInfo | None = None 

616) -> StreamingChunk: 

617 """ 

618 Converts the streaming response chunk from the OpenAI API to a StreamingChunk. 

619 

620 :param chunk: The chunk returned by the OpenAI API. 

621 :param previous_chunks: A list of previously received StreamingChunks. 

622 :param component_info: An optional `ComponentInfo` object containing information about the component that 

623 generated the chunk, such as the component name and type. 

624 

625 :returns: 

626 A StreamingChunk object representing the content of the chunk from the OpenAI API. 

627 """ 

628 finish_reason_mapping: dict[str, FinishReason] = { 

629 "stop": "stop", 

630 "length": "length", 

631 "content_filter": "content_filter", 

632 "tool_calls": "tool_calls", 

633 "function_call": "tool_calls", 

634 } 

635 # On very first chunk so len(previous_chunks) == 0, the Choices field only provides role info (e.g. "assistant") 

636 # Choices is empty if include_usage is set to True where the usage information is returned. 

637 if len(chunk.choices) == 0: 

638 return StreamingChunk( 

639 content="", 

640 component_info=component_info, 

641 # Index is None since it's only set to an int when a content block is present 

642 index=None, 

643 finish_reason=None, 

644 meta={ 

645 "model": chunk.model, 

646 "received_at": datetime.now().isoformat(), 

647 "usage": _serialize_object(chunk.usage), 

648 }, 

649 ) 

650 

651 choice: ChunkChoice = chunk.choices[0] 

652 

653 # create a list of ToolCallDelta objects from the tool calls 

654 if choice.delta and choice.delta.tool_calls: 

655 tool_calls_deltas = [] 

656 for tool_call in choice.delta.tool_calls: 

657 function = tool_call.function 

658 tool_calls_deltas.append( 

659 ToolCallDelta( 

660 index=tool_call.index, 

661 id=tool_call.id, 

662 tool_name=function.name if function else None, 

663 arguments=function.arguments if function and function.arguments else None, 

664 ) 

665 ) 

666 return StreamingChunk( 

667 content=choice.delta.content or "", 

668 component_info=component_info, 

669 # We adopt the first tool_calls_deltas.index as the overall index of the chunk. 

670 index=tool_calls_deltas[0].index, 

671 tool_calls=tool_calls_deltas, 

672 start=tool_calls_deltas[0].tool_name is not None, 

673 finish_reason=finish_reason_mapping.get(choice.finish_reason) if choice.finish_reason else None, 

674 meta={ 

675 "model": chunk.model, 

676 "index": choice.index, 

677 "tool_calls": choice.delta.tool_calls, 

678 "finish_reason": choice.finish_reason, 

679 "received_at": datetime.now().isoformat(), 

680 "usage": _serialize_object(chunk.usage), 

681 }, 

682 ) 

683 

684 # On very first chunk the choice field only provides role info (e.g. "assistant") so we set index to None 

685 # We set all chunks missing the content field to index of None. E.g. can happen if chunk only contains finish 

686 # reason. 

687 if choice.delta and (choice.delta.content is None or choice.delta.role is not None): 

688 resolved_index = None 

689 else: 

690 # We set the index to be 0 since if text content is being streamed then no tool calls are being streamed 

691 # NOTE: We may need to revisit this if OpenAI allows planning/thinking content before tool calls like 

692 # Anthropic Claude 

693 resolved_index = 0 

694 

695 # Initialize meta dictionary 

696 meta = { 

697 "model": chunk.model, 

698 "index": choice.index, 

699 "tool_calls": choice.delta.tool_calls if choice.delta and choice.delta.tool_calls else None, 

700 "finish_reason": choice.finish_reason, 

701 "received_at": datetime.now().isoformat(), 

702 "usage": _serialize_object(chunk.usage), 

703 } 

704 

705 # check if logprobs are present 

706 # logprobs are returned only for text content 

707 logprobs = _serialize_object(choice.logprobs) if choice.logprobs else None 

708 if logprobs: 

709 meta["logprobs"] = logprobs 

710 

711 content = "" 

712 if choice.delta and choice.delta.content: 

713 content = choice.delta.content 

714 

715 return StreamingChunk( 

716 content=content, 

717 component_info=component_info, 

718 index=resolved_index, 

719 # The first chunk is always a start message chunk that only contains role information, so if we reach here 

720 # and previous_chunks is length 1 then this is the start of text content. 

721 start=len(previous_chunks) == 1, 

722 finish_reason=finish_reason_mapping.get(choice.finish_reason) if choice.finish_reason else None, 

723 meta=meta, 

724 )
